# react-dashboard-ui

## Purpose
Implement practical dashboard screens and workflows.

## When To Use
Use for admin tools, operational dashboards, queues, charts, filters, and data-review UI.

## Inputs
User workflow, data contract, design system, target breakpoints, states, and acceptance criteria.

## Procedure
Inspect existing components and routes, build the main usable workflow first, include loading/empty/error states, ensure text fits, and verify desktop/mobile behavior.

## Safety Rules
Do not put PHI in fixtures, screenshots, local storage, or analytics events.

## Validation
Run build/tests or capture screenshots when the stack supports it.

## Output Format
User-visible changes, components touched, responsive/accessibility notes, and verification.
